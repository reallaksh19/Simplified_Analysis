# W10.R3 — Settings Authority Recovery

## Runtime authority

W10.R3 introduces one framework-neutral current-runtime settings owner and the closed contracts:

```text
engineering-settings-profile/v1
engineering-settings-audit/v1
settings-review-model/v1
workspace-consumer-registry/v7
application-view-state/v7
```

Registry and application view-state versions `v1` through `v6` retain their original canonical outputs. `SETTINGS` becomes `IMPLEMENTED` only in `v7`; all 11 established application identities and their navigation order are preserved.

## Active definition

| Setting | Type | Approved default | Allowed values | Consumer | Invalidation target |
|---|---|---|---|---|---|
| `reportTimestampPolicy` | enum | `exclude-from-deterministic-hash` | `exclude-from-deterministic-hash`, `include-in-export-content` | `REPORTS` | future `modelCalculationExportArtifact` only |

The default preserves the exact existing deterministic export artifact. The alternative appends a generated UTC timestamp only when the user later requests an export. It does not mutate the archived package, report, ledger or historical evidence.

## Transaction boundary

```text
immutable active profile
+ editable proposed values
→ validate complete proposal
→ compute dependency evidence
→ create immutable next profile and audit
→ persist one canonical profile
→ publish one engineeringSettings:changed event
```

Typing, selecting values, opening Settings, closing Settings, resetting the proposal or resetting the proposal to approved defaults does not alter the active profile. Invalid Apply preserves the active and persisted profiles, publishes no accepted change event and retains proposed values for correction.

## Persistence

```text
key: simplified-analysis:engineering-settings:v1
schema: engineering-settings-profile/v1
fields: schema, profileId, semanticHash, settings
```

Malformed, unsupported or stale storage is rejected. Approved defaults become active with a visible audit diagnostic; rejected storage is not overwritten until a later explicit valid Apply.

## Selective invalidation

`engineering-settings-audit/v1` records changed setting IDs, affected consumers and affected evidence. The active setting affects only a future export artifact. Existing model, calculation, report, package, session and ledger evidence is classified as unaffected and is never rewritten. No calculation, rebuild, export or dataset mutation runs automatically.

## Public read-only evidence

The Workspace public surface adds:

```text
getEngineeringSettingsProfile()
getEngineeringSettingsAudit()
getSettingsReviewModel()
```

No setter is exposed. All mutation enters through validated settings request events.

## Scope exclusions

W10.R3 does not change formulas, solver tolerances, engineering databases, PCF semantics, Sketcher, QA, Debug, LFEA/LAFEA, dependencies or `package-lock.json`. It does not delete the legacy settings files or remove React/Zustand dependencies.
