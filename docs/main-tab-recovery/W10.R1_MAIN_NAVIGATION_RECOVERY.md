# W10.R1 — Main Navigation Recovery Foundation

## Runtime boundary

The shipped entrypoint remains `src/main.js` and continues to call `bootstrapAnalysisWorkspace()`. The recovered application uses the existing framework-neutral `ApplicationShellController`, one `WorkspaceState`, the existing analysis session and ledger owners, and the existing single viewport host.

W10.R1 does not mount React, Zustand, `App.jsx`, `TopNav.jsx`, `Viewer3DTab`, or the legacy Sketcher canvas.

## Successor contracts

The runtime introduces:

- `workspace-consumer-registry/v5`
- `application-view-state/v5`
- `workspace-home-source/v1`
- `workspace-home-review-model/v1`

Registry serialization remains canonical by consumer identity. Navigation order is maintained separately.


## Home

Home is framework neutral and owns no mutable store. Its source and review model are materialized only after Home is opened. The source retains application status, dataset status and identity, and tab availability without retaining selected-entity identity or Workspace version.

Home provides quick navigation to available current-runtime consumers and explicitly identifies PCF, Sketcher, Settings, QA, and Debug as not yet migrated.

## Unavailable navigation

All 11 buttons remain visible and keyboard reachable. Unavailable activation leaves the current view unchanged and publishes `applicationView:changeFailed` with:

- `VIEW_RECOVERY_PENDING` for PCF, Sketcher, and Settings;
- `VIEW_NOT_IMPLEMENTED` for QA and Debug;
- `VIEW_NOT_AVAILABLE` for an implemented consumer lacking required current evidence.

The failure reason is shown in the navigation status output. No unavailable activation mounts legacy code or falls back to Workspace. Dataset replacement separately retains the existing policy of returning the application to Workspace.

## Scope boundary

This increment contains the runtime recovery only. It adds no dedicated source guards, workflow gates, certification jobs, test-registration changes, Local Stress integration, Element FEA integration, W11 work, PCF editing, Sketcher editing, mutable Settings, QA implementation, or Debug implementation.
