# W10.R4 — Framework-Neutral Sketcher Core Recovery

## Authority and contracts

W10.R4 adds one bounded `SketcherDraftAuthority` and the closed contracts:

```text
sketcher-draft-document/v1
sketcher-command/v1
sketcher-draft-audit/v1
sketcher-review-model/v1
sketcher-workspace-adoption/v1
workspace-consumer-registry/v8
application-view-state/v8
```

Registry and view-state versions `v1` through `v7` retain their canonical outputs. The exact 11-view order remains unchanged. `SKETCHER` becomes `IMPLEMENTED` only in `v8`, requires no dataset contract, and uses `DRAFT_GEOMETRY_AND_EXPLICIT_WORKSPACE_ADOPTION_ONLY`.

## Ownership

```text
ApplicationShellController = application view-state owner
WorkspaceState             = active dataset authority
DatasetController          = Workspace dataset-loading boundary
SketcherDraftAuthority     = bounded immutable draft transaction owner
SettingsAuthority          = active settings authority
EventBus                   = request/outcome boundary
```

The Sketcher controller never imports or mutates `WorkspaceState`, never changes application navigation directly, and never starts analysis. UI selection, pointer, hover, pan, zoom, focus and in-progress drawing state remain outside the canonical document.

## Draft semantics

The canonical document is millimetre-based and supports `XY`, `XZ`, and `YZ` working planes. W10.R4 produces straight `PIPE` centerlines only. Nodes and segments are canonically sorted by identity; all published documents, commands, audits, review models and adoption evidence are plain, deeply frozen and semantically hashed.

Deterministic identities use bounded sequences:

```text
draft-001
N001
S001
command-000001
```

No clock, random, UUID, persistence, DOM or browser input participates in canonical identity. Connections require explicit shared node identity. Grid snapping is an explicit command option and no proximity merge is inferred.

Accepted edit commands validate a complete candidate before replacing the active document. Rejected commands preserve the document and both history stacks. Undo and redo restore exact document snapshots and semantic hashes. Referenced-node deletion uses `REJECT_WHEN_REFERENCED`.

Loading an exported canonical JSON document is an exact authority replacement and preserves its semantic hash. Workspace import is separately classified as a derived new draft revision.

## Topology and adoption policy

The deterministic topology audit reports:

```text
CONNECTED_COMPONENT_COUNT
ISOLATED_NODE
DANGLING_ENDPOINT
BRANCH_NODE
CLOSED_LOOP
ZERO_LENGTH_SEGMENT
DUPLICATE_SEGMENT
MISSING_ENDPOINT
NONFINITE_COORDINATE
```

Branching, disconnected components and closed loops are visible but non-blocking. Invalid geometry, invalid identity/reference evidence, an empty draft, and non-full-fidelity Workspace imports block adoption.

Explicit adoption follows only this path:

```text
immutable draft
→ inputxml-managed-stage/v1 package
→ normalizeWorkspaceDataset proof
→ shared-model and topology preparation
→ coordinate/count parity proof
→ dataset:loadRequested
→ DatasetController
→ WorkspaceState
```

One draft segment produces exactly one adopted pipe. A failed qualification or dataset load preserves both the active draft and active Workspace dataset. A successful adoption publishes one load request, retains immutable `sketcher-workspace-adoption/v1` evidence and relies on the existing dataset-replacement navigation behavior. It starts no solver, screening, FEA, report or export action.

## Workspace import

The pure import adapter consumes only the current `sharedModel` and optional `topologyGraph`. Supported straight pipe centerlines become deterministic nodes and segments. Node unification uses explicit topology connections, never coordinate proximity. Unsupported entities and unavailable topology are disclosed and classify the result as `PARTIAL_WITH_DIAGNOSTICS`; such drafts remain reviewable and exportable but are not adoption eligible.

## Framework-neutral view

The shipped view uses ordinary DOM and SVG. It provides blank drafting, optional Workspace import, deterministic JSON load/save, pipe drawing, exact-node reuse, node movement, explicit deletion, undo/redo, validation, fit, pan, zoom, pointer coordinates, selection properties, semantic evidence and adoption status. Source-derived content is assigned through DOM text APIs.

The Sketcher runtime imports no React, ReactDOM, Zustand, R3F, Drei, Three.js, legacy application store or analysis store. The application retains exactly one `data-webgl-host`; Sketcher creates no canvas or WebGL renderer.

## Deferred capabilities

W10.R4 does not migrate the React/R3F Sketcher, Master DB insertion, valve/flange placement, bend/tee/olet tools, component recognition, PCFX authoring UI, advanced dimensions, marquee editing, support/restraint placement, 2D or 3D analysis routing, temperature-driven calculation, automatic solver execution, or persistence. Those capabilities require separate authority and engineering work packs.

## Source and dependency boundary

Production changes are limited to `src/core/sketcher-draft/**`, `src/workspace/sketcher-*.js`, additive `v8` consumer evolution, and the minimum shell/layout/bootstrap/event wiring. Supporting files are limited to W10.R4 scripts, browser certification, documentation, workflow and package command registration. `package-lock.json`, dependencies, PCF semantics, Settings semantics, Reports semantics, Pipe Solver formulas, LFEA mechanics and LAFEA mechanics are unchanged.
