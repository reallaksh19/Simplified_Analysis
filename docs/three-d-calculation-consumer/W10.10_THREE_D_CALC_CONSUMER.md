# W10.10 — 3D Calc Evidence Consumer

## Scope

W10.10 adopts existing immutable engineering evidence in the shipped framework-neutral application shell. It does not introduce a solver, renderer, viewport, coordinate transformation, formula, support authoring workflow or engineering override.

```text
workspace-consumer-context/v1
→ exact W10.1 shared model
→ exact W10.2 topology
→ exact W10.3 support attachment and restraint capability
→ optional W10.4 model-load primitives
→ optional complete W10.6 vertical-beam evidence
→ three-d-calculation-review-model/v1
→ THREE_D_CALC view
```

## Required evidence

The review model requires one linked dataset containing:

- `shared-piping-model/v1`;
- `piping-port-topology-graph/v1`;
- `topology-connection-audit/v1`;
- `support-attachment-model/v1` and audit;
- `restraint-capability-model/v1` and audit.

Every required contract is validated with its official validator. Shared-model, topology, attachment and restraint semantic links must agree exactly.

## Optional evidence

`model-load-primitive-set/v1` is accepted only when the consumer context has already retained its linked load-case evidence. Partial, stale or wrong-dataset evidence is excluded and diagnosed.

Vertical-beam evidence is accepted only as the complete linked set:

```text
flexural-property-projection/v1
vertical-beam-model/v1
vertical-beam-solution/v1
vertical-beam-solver-audit/v1
```

The view preserves signed support force, displacement, rotation and residual evidence exactly. Its scope is **scalar linear-elastic vertical Euler–Bernoulli beam evidence**, not general three-dimensional piping flexibility.

## Contract evolution

W10.8 and W10.9 contracts remain unchanged:

```text
workspace-consumer-registry/v1
workspace-consumer-registry/v2
application-view-state/v1
application-view-state/v2
```

W10.10 adds:

```text
workspace-consumer-registry/v3
application-view-state/v3
```

Only Workspace, Reports, Load Calc and 3D Calc are implemented in v3. Pipe Solver, QA and Debug remain `NOT_IMPLEMENTED`.

## Action boundary

The consumer delegates only these existing explicit requests:

```text
sharedModel:exportRequested
topology:rebuildExactRequested
topology:exportRequested
supportRestraint:rebuildEvidenceRequested
supportRestraint:exportRequested
verticalBeam:rebuildRequested
verticalBeam:solveRequested
verticalBeam:exportRequested
```

Action eligibility is derived independently from validated prerequisites. Existing controllers remain the only builders, solver and exporters.

## Explicit limitations

- Read-only evidence review.
- Not a second 3D viewport.
- Not general 3D piping flexibility.
- No thermal or pressure stress.
- No nonlinear spring, gap or contact solution.
- No horizontal-force or code-stress qualification.
- Vertical-beam evidence is scalar and topology-local.
