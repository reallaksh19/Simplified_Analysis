# W10.8 Framework-Neutral Workspace Consumers

## Scope

W10.8 introduces a read-only consumer boundary over the immutable W10.1–W10.7 engineering contracts. It does not execute screening, beam solving, package creation, or export automatically.

## Contracts

- `workspace-consumer-context/v1` retains exact validated source-contract references or `null`.
- `workspace-consumer-registry/v1` declares implemented and future consumers without treating dormant React components as runtime implementations.
- `workspace-consumer-readiness/v1` reports availability, missing contracts, invalid contracts, or non-implementation. Readiness is not an engineering qualification.
- `application-view-state/v1` controls the implemented `WORKSPACE` and `REPORTS` views.

The context validates public W10.1–W10.7 contracts and their adjacent semantic links. Invalid, stale, or dataset-mismatched evidence is excluded and diagnosed; it is never repaired or re-hashed.

## Consumer registry

| Consumer | Status | Current behavior |
| --- | --- | --- |
| Workspace | Implemented | Existing three-panel model and explicit analysis workflow |
| Reports | Implemented | Selected W10.7 archived package/report review and export |
| Load Calc | Not implemented | Disabled declaration only |
| 3D Calc | Not implemented | Disabled declaration only |
| Pipe Solver | Not implemented | Disabled declaration only |
| QA | Not implemented | Disabled declaration only |
| Debug | Not implemented | Disabled declaration only |

## Reports adoption

Reports renders only the active validated `model-calculation-package/v1` and matching `model-calculation-report/v1` selected through the W10.7 ledger. It preserves W10.5 `screenedVerticalForceN` terminology and W10.6 `signedSupportForceN` / `upwardSupportForceN` terminology.

Archived-entry selection and JSON, CSV, and Markdown exports delegate to the existing W10.7 events. Reports contains no solver or package-creation calls and explicitly states that the output is not a full pipe-stress or code-compliance report.

## State and lifecycle

The application shell hides and shows retained view containers. It does not recreate the Workspace or its controllers. Dataset, selection, topology, restraints, loads, screening, beam results, and calculation archive therefore remain intact during valid view changes.

When Reports readiness is lost, the view state returns to Workspace. Dataset replacement, dataset clear, and teardown retain the existing W10.1–W10.7 lifecycle rules and clean up W10.8 listeners and state.

## Public API

```text
AnalysisWorkspace.getWorkspaceConsumerContext()
AnalysisWorkspace.listWorkspaceConsumers()
AnalysisWorkspace.getWorkspaceConsumerReadiness(consumerId)
AnalysisWorkspace.getApplicationViewState()
AnalysisWorkspace.activateApplicationView(viewId)
```

All returned contract values are deeply immutable. No mutable consumer store is exposed.
