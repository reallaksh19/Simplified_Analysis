# W10.8 Workspace Consumer Boundary

## Scope

W10.8 adds a framework-neutral consumer boundary to the shipped Analysis Workspace runtime. It does not mount React or Zustand and does not add an engineering calculation.

## Contracts

- `workspace-consumer-context/v1` retains exact immutable W10.1–W10.7 contract references or `null`.
- `workspace-consumer-registry/v1` declares the closed consumer set.
- `workspace-consumer-readiness/v1` reports application-input availability, not engineering qualification.
- `application-view-state/v1` controls the implemented `WORKSPACE` and `REPORTS` views.

The context preserves source semantic hashes. Dataset-mismatched, invalid, or stale archive evidence is excluded and diagnosed rather than repaired.

## Consumers

`WORKSPACE` and `REPORTS` are implemented. `LOAD_CALC`, `THREE_D_CALC`, `PIPE_SOLVER`, `QA`, and `DEBUG` remain explicitly `NOT_IMPLEMENTED` and disabled in navigation.

## Reports

Reports renders the selected immutable W10.7 ledger entry, package, and report. Archived-entry selection and JSON, CSV, and Markdown export use the existing W10.7 event boundary. Reports does not run screening, solve a beam, create a package, or download automatically.

The report remains limited to topology-local tributary vertical-load screening and scalar linear-elastic vertical beam evidence. It is not a full pipe-stress or code-compliance report.

## State policy

View changes hide and show existing containers. They do not rebuild controllers or reset dataset, entity selection, topology, supports, model loads, screening, beam solutions, or model-calculation history. Dataset replacement or clear returns safely to Workspace when Reports evidence is unavailable.

## Public API

- `AnalysisWorkspace.getWorkspaceConsumerContext()`
- `AnalysisWorkspace.listWorkspaceConsumers()`
- `AnalysisWorkspace.getWorkspaceConsumerReadiness(consumerId)`
- `AnalysisWorkspace.getApplicationViewState()`
- `AnalysisWorkspace.activateApplicationView(viewId)`

All returned contract values are read-only. Navigation cannot execute calculations.
