# W10.11 — Guarded Pipe Solver Consumer

## Scope

W10.11 exposes the existing `pipe-screening` capability in the shipped framework-neutral shell. It does not create, copy or modify a solver, session store, coordinator, ledger store, engineering formula or result contract.

```text
workspace-consumer-context/v1
+ selected Workspace entity
+ existing capability inspection
+ existing analysis-session/v1
+ existing analysis-ledger/v1 entries
→ pipe-solver-consumer-source/v1
→ pipe-solver-review-model/v1
→ PIPE_SOLVER view
```

## Source snapshot

The Workspace adapter receives the existing Workspace state, capability registry, analysis-session store and analysis-ledger store by explicit injection. It never constructs replacement owners. The source contract retains:

- current dataset, workspace version and consumer-context semantic hash;
- selected entity identity and exact source-attribute/native-parameter references;
- exact registered `pipe-screening` inspection/readiness;
- only the matching active session;
- only current-dataset Pipe Solver ledger entries;
- deterministic exclusion diagnostics for stale or mismatched evidence.

Source construction performs no capability execution and no state mutation.

## Review model

`pipe-solver-review-model/v1` is a deterministic read-only projection. It retains the exact source snapshot and projects:

- capability manifest and separate applicability/review/run readiness;
- input fields, units, provenance, reviewed overrides and owner-produced field errors;
- exact session lifecycle evidence;
- exact accepted `solver-result-contract-v1` evidence;
- dataset-scoped Pipe Solver ledger history;
- assumptions, limitations and diagnostics.

Completed results are displayed only when they remain:

```text
schemaVersion    solver-result-contract-v1
engineeringLevel BENCHMARKED_SCREENING
methodId          SIMPLIFIED_2D_TOPOLOGY_SCREENING
```

Malformed or unexpected completed results are excluded and diagnosed, never repaired.

## Readiness boundaries

Generic application readiness requires only the immutable shared model, topology graph and topology audit. Selection applicability and execution readiness remain inside the Pipe Solver review model and are owned by the existing registered capability inspection.

## Existing action boundary

The consumer publishes only:

```text
analysis:sessionOpenRequested
analysis:sessionOverrideRequested
analysis:sessionResetRequested
analysis:sessionCloseRequested
analysis:requested
analysis:ledgerActiveRequested
analysis:exportRequested
```

Every action is explicitly initiated and guarded against exact current evidence. Existing controllers remain the sole owners of override normalization, session mutation, analysis execution, ledger selection and artifact export.

## Explicit limitations

- Benchmarked simplified 2D screening only.
- Not final piping-code stress analysis.
- Not general 3D piping flexibility.
- No branch flexibility or branch SIF solution.
- No equipment interaction.
- No nonlinear gaps, friction, contact or restraint stiffness.
- No automatic execution.
- Inputs must be explicit or user-overridden through the existing session boundary.
