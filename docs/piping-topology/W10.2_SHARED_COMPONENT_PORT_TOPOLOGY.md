# W10.2 Shared Component-Port Topology

## Scope

Mission W10.2 builds deterministic component-port connectivity from `shared-piping-model/v1` only. It adds reusable topology inspection and explicit Workspace export without introducing routes, support attachment, restraints, load cases, load distribution, calculations, renderer changes, or authoring behavior.

## Contracts

The mission adds four deeply immutable contracts:

- `topology-connection-profile/v1`
- `engineering-port-projection/v1`
- `piping-port-topology-graph/v1`
- `topology-connection-audit/v1`

The topology graph is framework-neutral and has no DOM, React, Zustand, Three.js, solver, report, or Workspace dependency.

## Evidence precedence

Connections are resolved in strict stages:

```text
EXPLICIT_CONNECTION_REFERENCE
→ SHARED_SOURCE_ENDPOINT_IDENTITY
→ EXACT_COORDINATE
→ TOLERANCE_INFERRED
```

Once a valid higher-precedence connection consumes a normal port, lower stages cannot replace it. Multiple plausible candidates remain ambiguous and unconnected. Invalid self-connections and inferred same-component connections are rejected and audited.

## Exact-only default

`EXACT_ONLY_V1` is the default profile. It permits explicit references, shared endpoint identity and exact coordinate equality. It does not include a tolerance value and never emits `TOLERANCE_INFERRED`.

Exact coordinate matching uses canonical unit normalization when the shared model declares a supported length unit. With unknown units, exact numerical equality within the single model remains usable because zero distance is unit invariant; tolerance inference is blocked.

## Explicit tolerance profile

`TOLERANCE_EXPLICIT_V1` requires a visible positive tolerance. The profile records the tolerance and its unit. Supported units are converted deterministically to millimetres for physical comparison.

Tolerance candidates are found with a 3D spatial hash. Each port inspects its current cell and 26 neighbouring cells. The production algorithm does not compare every port with every other port.

Unknown model units block tolerance inference with `UNIT_BLOCKED` evidence. No millimetre assumption or project default is injected.

## Identity compatibility

Identity comparison is separate from physical distance:

- matching non-empty line or system identity increases dimensionless confidence;
- conflicting non-empty system identity blocks inferred connectivity;
- conflicting non-empty line identity requires explicit evidence;
- empty identity is neutral, not a match;
- identity status is retained on every connection and in rejection diagnostics.

Explicit source evidence may preserve a connection despite identity conflict, while retaining the conflict diagnostics and physical distance when calculable.

## Multi-port components

Tee and cross records remain components with three or four distinct ports. The topology graph joins external peers to those ports; it does not collapse the fitting into one start/end edge. A port has one physical peer unless its source reference explicitly declares multi-connection behavior.

## Connected components and cycles

Connected components are built deterministically from component-port connections. Stable IDs derive from sorted component keys. Cycles are detected as topology evidence and reported through `cyclicComponentIds`; a valid closed piping loop is not treated as an error.

## Workspace capability

The current right panel displays:

- components;
- ports;
- connections;
- connected components;
- unconnected ports;
- ambiguous ports;
- cycles;
- active profile.

It provides explicit actions:

- `Rebuild Exact Topology`
- `Rebuild With Tolerance`
- `Export Topology Graph`

Exact topology is built after shared-model import. Tolerance inference requires explicit activation. Invalid tolerance input leaves the prior valid graph intact. Export is deterministic UTF-8 JSON with a dataset-derived filename and immediate object-URL revocation.

Read-only diagnostics are exposed through:

```text
AnalysisWorkspace.getTopologyGraph()
AnalysisWorkspace.getTopologyAudit()
```

Clear and teardown remove the graph and all topology listeners. No calculation is executed automatically.

## Known limitations

- Connections are limited to evidence already present in `shared-piping-model/v1`.
- Missing or ambiguous evidence remains unconnected with deterministic diagnostics.
- Permanent project tolerance calibration is not provided.
- Support attachment, restraints, routes, chainage, load paths, load cases and solver behavior remain future Work Packs.
