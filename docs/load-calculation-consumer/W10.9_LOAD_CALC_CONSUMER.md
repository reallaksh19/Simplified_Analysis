# W10.9 — Reusable Load Calc Consumer Adoption

## Scope

W10.9 adopts existing W10.4 model-load evidence as a real framework-neutral Load Calc application view. It optionally displays complete linked W10.5 topology-local tributary screening evidence.

```text
validated workspace-consumer-context/v1
        │
        ├── load-case-set/v1
        ├── model-load-primitive-set/v1
        ├── model-load-readiness-audit/v1
        ├── optional complete W10.5 evidence
        │
        └── load-calculation-review-model/v1
                    │
                    └── LOAD_CALC current-runtime view
```

The review model does not calculate, solve, distribute, infer, default, convert, select governing cases or reinterpret engineering meaning.

## Contract evolution

W10.8 contracts remain available unchanged:

- `workspace-consumer-registry/v1`
- `application-view-state/v1`

W10.9 adds explicit successors:

- `workspace-consumer-registry/v2`
- `application-view-state/v2`

Registry v1 still declares Load Calc `NOT_IMPLEMENTED`. Registry v2 declares Workspace, Reports and Load Calc `IMPLEMENTED`. Three-dimensional calculation, Pipe Solver, QA and Debug remain `NOT_IMPLEMENTED`.

The generic `workspace-consumer-context/v1` and `workspace-consumer-readiness/v1` semantics remain unchanged. Readiness records are reconstructed against the exact registry version supplied by the application shell.

## Review-model evidence

`load-calculation-review-model/v1` retains:

- exact source semantic hashes;
- exact validated source context by immutable reference;
- W10.4 load-case definitions and readiness-audit totals;
- component outcomes without inferred modes;
- distributed, point and explicit-moment primitives;
- exact geometry, null vector evidence, mass evidence and force evidence;
- formula traces in authored provenance order;
- optional W10.5 path/case screening audit rows;
- deterministic limitations, diagnostics and summary.

The contract identity excludes selected rows, navigation state, status messages and active subtabs. It uses no time, UUID, randomness or mutable sequence identity.

## W10.4 terminology

Load-case qualification remains exactly:

- `READY`
- `BLOCKED`

Totals come only from `model-load-readiness-audit/v1`. The Load Calc consumer does not recalculate total mass or force.

## W10.5 terminology

The visible section is titled **Topology-local tributary screening**.

`screenedVerticalForceN` remains tributary-screening evidence. It is not renamed as a general support reaction, stress reaction, stiffness reaction or beam reaction.

Incomplete, stale, mixed-dataset or mismatched optional W10.5 evidence is excluded from the review model and represented by deterministic diagnostics.

## Action boundary

Load Calc can publish only existing requests:

- `modelLoad:rebuildRequested`
- `modelLoad:exportRequested`
- `supportLoadScreening:rebuildPathsRequested`
- `supportLoadScreening:runRequested`
- `supportLoadScreening:exportRequested`

Existing W10.4 and W10.5 controllers remain sole calculation and export owners. View activation emits no calculation request and performs no automatic export.

## Application shell

The shipped entrypoint remains `src/main.js → bootstrapAnalysisWorkspace()`.

The application shell keeps the three-panel Workspace mounted and adds a retained Load Calc container. Navigation uses the W10.8 focusable `aria-disabled` pattern. View switching does not re-bootstrap controllers or mutate engineering stores.

Public API:

```text
AnalysisWorkspace.getLoadCalculationReviewModel()
```

The getter returns the current deeply immutable validated review model or `null`.

## Engineering limitations

The view prominently states:

- component gravity loads only;
- not full piping flexibility;
- no thermal or pressure stress;
- no horizontal restraint forces;
- no code-stress qualification;
- tributary screening is not stiffness reaction analysis.

No dormant React/Zustand calculation workspace, new load engine, support-reaction solver, vertical-beam solve, support authoring, 3D overlay, PDF or approval workflow is introduced.
