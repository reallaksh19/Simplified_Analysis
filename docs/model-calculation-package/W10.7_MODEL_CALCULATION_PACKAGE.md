# W10.7 — Model Calculation Package, Archive and Report Projection

## Scope

W10.7 packages already-completed W10.5 topology-local tributary screening and W10.6 scalar vertical Euler–Bernoulli solutions. It does not execute, modify or reinterpret either calculation.

```text
completed W10.5 screening snapshot
and/or completed W10.6 vertical-beam snapshot
→ model-calculation-package/v1
→ model-calculation-ledger/v1
→ model-calculation-report/v1
→ model-calculation-export-artifact/v1
```

This evidence is not a full piping stress calculation, code-compliance report or nozzle-load qualification.

## Contracts

- `model-calculation-package/v1`
- `model-calculation-ledger/v1`
- `model-calculation-ledger-entry/v1`
- `model-calculation-report/v1`
- `model-calculation-export-artifact/v1`

All contracts are deeply immutable. Package and report identities are derived from canonical semantic content without timestamps, UUIDs, random values or process-local sequence values.

## Package modes

- `TRIBUTARY_SCREENING_ONLY`
- `VERTICAL_BEAM_ONLY`
- `SCREENING_AND_VERTICAL_BEAM`

A package requires at least one completed explicit result. Readiness-only W10.5 or W10.6 state is not packageable.

## Stale and mixed-contract rejection

Screening packages validate the W10.5 profile, path model, screening result and audit, including dataset, profile, path-model and result/audit semantic links.

Beam packages validate the W10.6 profile, flexural projection, beam model, explicit solution and audit, including dataset, profile, path-model, projection, model, solution and audit semantic links.

Combined packages additionally require the same dataset, vertical path model, path/load-case identities, load-case set, load primitive set and model-load readiness audit.

## Terminology

W10.5 and W10.6 evidence remains distinct:

- `screenedVerticalForceN` is the tributary screening force.
- `signedSupportForceN` is the W10.6 positive-down signed support force.
- `upwardSupportForceN` is the nonnegative upward magnitude projected by W10.6.

A screened force is never renamed as a stiffness reaction.

## Method evidence

Included calculations retain method/profile identity, formula IDs, sign convention, assumptions, limitations, result hash and audit hash.

Engineering levels are:

- `BENCHMARKED_SCREENING`
- `LINEAR_ELASTIC_VERTICAL_BEAM`

The package explicitly excludes 3D piping flexibility, thermal expansion, pressure stress, spring/gap nonlinearities, horizontal restraint forces, code-stress qualification and nozzle-load qualification.

## Dataset-scoped archive

Packages are archived only through the explicit **Create Calculation Package** action. The separate model-calculation ledger:

- suppresses duplicate package semantic hashes;
- retains the latest 100 visible entries;
- uses deterministic dataset-local sequence IDs;
- supports active-entry selection;
- survives entity selection changes;
- resets on dataset replacement, dataset clear, explicit history clear and Workspace teardown.

Archived package objects remain immutable when upstream Workspace state changes.

## Reports and exports

Reports are built only from the selected ledger entry. Explicit exports are available as deterministic newline-terminated UTF-8:

- JSON containing validated package and report evidence;
- CSV normalized by package, method, path/case, screening force, beam support force, residual, blocker and diagnostic;
- Markdown with engineering levels, assumptions, limitations, qualification, support forces, displacement, rotation and residual evidence.

Each artifact retains exact UTF-8 byte length, a stable package-derived filename and immediate object-URL revocation. PDF is outside W10.7.

## Workspace behavior

Snapshot availability may update automatically. Package creation and all exports require explicit user activation. W10.7 never emits solver execution requests, `analysis:started`, W10.5 run requests or W10.6 solve requests.

Read-only APIs:

```text
AnalysisWorkspace.getModelCalculationLedger()
AnalysisWorkspace.getActiveModelCalculationPackage()
AnalysisWorkspace.getActiveModelCalculationReport()
```
