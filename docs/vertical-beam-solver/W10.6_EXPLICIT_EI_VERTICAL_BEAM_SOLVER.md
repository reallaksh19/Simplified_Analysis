# W10.6 — Explicit-EI Vertical Beam Solver

## Scope

W10.6 consumes the reusable W10.1, W10.4 and W10.5 contracts and produces a scalar, linear-elastic Euler–Bernoulli vertical beam solution.

```text
shared-piping-model/v1
+ vertical-load-path-model/v1
+ load-case-set/v1
+ model-load-primitive-set/v1
+ model-load-readiness-audit/v1
        │
        ├── vertical-beam-solver-profile/v1
        ├── flexural-property-projection/v1
        ├── vertical-beam-model/v1
        ├── vertical-beam-solution/v1
        └── vertical-beam-solver-audit/v1
```

The capability is not a 3D piping flexibility, thermal expansion, nonlinear support, spring/gap, code-stress or nozzle-load solution.

## Explicit flexural evidence

The shared model preserves only narrow, unit-qualified aliases:

- `elasticModulusMpa`: `ELASTIC_MODULUS_MPA`, `YOUNGS_MODULUS_MPA`, `YOUNG_MODULUS_MPA`, `MODULUS_OF_ELASTICITY_MPA`.
- `secondMomentAreaMm4`: `SECOND_MOMENT_AREA_MM4`, `AREA_MOMENT_OF_INERTIA_MM4`.
- `flexuralRigidityNm2`: `FLEXURAL_RIGIDITY_N_M2`, `EI_N_M2`.

The aliases `E`, `I`, `MODULUS` and `INERTIA` are intentionally not accepted. Material names and component types never supply a default Young’s modulus.

## Flexural precedence

The immutable `flexural-property-projection/v1` resolves each W10.5 component interval using:

```text
direct positive EI
→ explicit positive E × direct positive I
→ explicit positive E × derived circular-section I
```

Direct EI blocks fallback when present but invalid. Direct I blocks circular-section fallback when present but invalid. Only `PIPE`, `STRAIGHT_PIPE` and `TUBE` may use E/I or circular-section derivation; other component types require direct EI.

Canonical units are Pa, m⁴ and N·m². Circular hollow-section evidence uses:

```text
ID = OD − 2 × wall
I = π/64 × (OD⁴ − ID⁴)
EI = E × I
```

Formula traces retain source paths, source units, canonical substitutions and formula IDs. No temperature-dependent modulus is inferred.

## Beam model

`vertical-beam-model/v1` creates nodes only at evidence boundaries:

- path terminals;
- component boundaries;
- qualified support stations;
- exact point-load stations;
- distributed-load boundaries.

There is no arbitrary mesh density. Each node has a vertical displacement DOF and a rotation DOF. Each element has positive length, belongs to one component interval, references one resolved EI record and cannot cross an EI discontinuity.

Qualified W10.5 rigid supports constrain vertical translation to zero. Rotation is free. Constraints are applied by free/constrained DOF partitioning; no penalty stiffness is used. At least two distinct qualified supports are required. Free terminal overhangs remain part of the beam and may solve.

## Element and load formulas

The element stiffness formula is `EULER_BERNOULLI_ELEMENT_STIFFNESS_V1`:

```text
EI/L³ ×
[ 12,   6L, -12,   6L
  6L, 4L²,  -6L, 2L²
 -12,  -6L,  12,  -6L
  6L, 2L²,  -6L, 4L² ]
```

Uniform gravity load uses the consistent vector `EULER_BERNOULLI_UNIFORM_LOAD_VECTOR_V1` under the positive-down convention:

```text
[ wL/2, wL²/12, wL/2, -wL²/12 ]
```

Point force uses `EULER_BERNOULLI_NODE_POINT_LOAD_V1` and must have an exact mesh node. Point moments remain blocked because W10.4 does not certify a scalar bending-axis mapping.

## Linear solve

`VERTICAL_BEAM_GLOBAL_ASSEMBLY_V1` assembles a deterministic symmetric global matrix. `SCALED_PARTIAL_PIVOT_LINEAR_SOLVE_V1` solves the free partition using deterministic scaled partial pivoting with exposed absolute and relative pivot tolerances.

The solution retains matrix dimensions, free and constrained DOF counts, minimum pivot, minimum scaled pivot, solver trace and free-DOF residual. Non-finite, singular and near-singular systems are blocked without a numerical dependency.

## Sign convention and equilibrium

```text
vertical displacement positive = GRAVITY_DOWN
applied vertical force positive = GRAVITY_DOWN
support rotation = free
qualified rigid-support vertical displacement = 0
signed support force positive = GRAVITY_DOWN
```

`upwardSupportForceN = max(0, -signedSupportForceN)`. A positive-down support force is preserved and diagnosed as `SUPPORT_UPLIFT_OR_DIRECTION_REVERSAL`; it is not discarded or clamped.

Every ready path/case proves:

- `VERTICAL_BEAM_FORCE_EQUILIBRIUM_V1`;
- `VERTICAL_BEAM_MOMENT_EQUILIBRIUM_V1` about path station zero;
- `VERTICAL_BEAM_MATRIX_RESIDUAL_V1` on free DOFs;
- rigid-support displacement against the exposed tolerance.

Engineering values are not rounded to force equilibrium.

## Workspace lifecycle

The Vertical Beam card may rebuild readiness automatically after approved upstream changes. Numerical solution and export require explicit user actions:

```text
Rebuild Vertical Beam Model
Solve Vertical Stiffness
Export Vertical Beam Solution
```

Upstream changes clear stale solutions while retaining the latest valid rebuilt model. Export is deterministic UTF-8 JSON with a dataset-derived filename and immediate object-URL revocation. Dataset, selection, Workspace version and upstream semantic hashes are not modified.

Read-only APIs:

```text
AnalysisWorkspace.getFlexuralPropertyProjection()
AnalysisWorkspace.getVerticalBeamModel()
AnalysisWorkspace.getVerticalBeamSolution()
AnalysisWorkspace.getVerticalBeamSolverAudit()
```

Clear and teardown remove state and listeners.

## Certified limitations

- Simple W10.5 acyclic non-branching paths only.
- Scalar vertical Euler–Bernoulli bending only.
- Linear elastic, small-displacement response.
- Explicit EI evidence only; no material default.
- No point-moment axis inference.
- No spring, gap, settlement or rotational restraint.
- No branch/loop load distribution.
- No 3D bending plane, thermal, pressure, lateral or nonlinear behavior.
- No code-stress or nozzle-load assessment.
