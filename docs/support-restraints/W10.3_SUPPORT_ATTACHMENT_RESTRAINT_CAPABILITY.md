# W10.3 Shared Support Attachment and Restraint Capability

## Scope

Mission W10.3 builds deterministic support attachment and semantic restraint capability from `shared-piping-model/v1` and `piping-port-topology-graph/v1` only.

It does not add routes, chainage, load cases, load primitives, reactions, solver mathematics, global restraint vectors, viewport changes, authoring, or reporting behavior.

## Contracts

The mission adds seven deeply immutable contracts:

- `support-attachment-profile/v1`
- `engineering-support-projection/v1`
- `support-attachment-model/v1`
- `support-attachment-audit/v1`
- `restraint-classification-profile/v1`
- `restraint-capability-model/v1`
- `restraint-capability-audit/v1`

All identities and semantic hashes are deterministic. No timestamp or random identity is used.

## Support evidence

The shared-model Workspace adapter preserves optional explicit support evidence for attachment references, support type, semantic directional capabilities, gaps, stiffness, spring rate, friction, and multi-attachment declarations.

Each retained value includes its source kind, source path, and declared unit. Conflicting source values remain visible as conflict diagnostics. Missing values remain missing.

The W10.3 core never traverses the raw source snapshot.

## Attachment profiles

`EVIDENCE_ONLY_V1` is the default. It permits only explicit port references, explicit component references, and a unique source parent/path relationship.

`GEOMETRIC_PROJECTION_EXPLICIT_V1` requires explicit activation, a positive visible tolerance, and a supported model length unit. Unknown units block projection. No tolerance is inferred or defaulted.

## Attachment precedence

Attachment resolution follows strict precedence:

```text
EXPLICIT_ATTACHED_PORT_REFERENCE
→ EXPLICIT_ATTACHED_COMPONENT_REFERENCE
→ UNIQUE_SOURCE_PARENT_OR_PATH_RELATION
→ GEOMETRIC_PROJECTION_EXPLICIT
```

Identity evidence may filter candidates, but identity alone cannot attach a support. Conflicting system identity blocks inferred attachment. Conflicting line identity requires explicit evidence.

Explicit evidence may retain an attachment despite a coordinate offset. Physical distance and diagnostics remain visible.

## Attachment targets

Approved targets are limited to:

- component port points;
- explicit component center points;
- two-port component centerline segments;
- explicit-center-to-port legs for multi-port components.

The implementation never creates arbitrary tee or cross chords. Geometry absent from the shared model is not invented.

Geometric projection uses deterministic clamped point-to-segment projection. Candidate lookup uses a spatial segment index and the current cell plus 26 neighbours rather than an all-supports-by-all-targets scan.

Ambiguous equal candidates remain unattached.

## Restraint classification

`EXPLICIT_AND_TYPE_CLASSIFICATION_V1` uses semantic directions:

```text
VERTICAL
LATERAL
LONGITUDINAL
ROTATIONAL
```

Closed direction states are:

```text
RESTRAINED
FREE
GAP
SPRING
UNKNOWN
CONFLICT
```

Explicit directional evidence has precedence. Support-type mapping is identified as `INFERRED_FROM_SUPPORT_TYPE`, never explicit source evidence.

Closed qualifications are:

```text
EXPLICITLY_RESOLVED
TYPE_CLASSIFIED
PARTIALLY_RESOLVED
UNRESOLVED
BLOCKED_ATTACHMENT
CONFLICTED
```

Unattached supports remain blocked for future solver projection. Springs without explicit stiffness remain `SPRING` with unresolved stiffness. No gap, stiffness, spring rate, friction, or capability defaults are introduced.

## Workspace capability

The Analysis Workspace support/restraint card displays attachment and restraint health, active profiles, and explicit rebuild actions:

- `Rebuild Evidence Attachments`
- `Rebuild With Projection`
- `Export Support/Restraint Model`

Evidence-only models rebuild after shared-model or topology changes. Projection requires explicit activation. Invalid tolerance retains the prior valid models.

Export is deterministic UTF-8 JSON, uses a dataset-derived filename, revokes object URLs, and preserves Workspace dataset, selection, shared model, topology graph, and version.

Read-only getters expose the current immutable models and audits.

## Known limitations

- Attachment is limited to evidence and geometry already preserved by W10.1/W10.2.
- Insufficient or ambiguous evidence remains unattached.
- No permanent projection-tolerance calibration is included.
- Semantic restraints do not provide global X/Y/Z vectors because no certified local support frame exists.
- No support authoring or manual override is included.
