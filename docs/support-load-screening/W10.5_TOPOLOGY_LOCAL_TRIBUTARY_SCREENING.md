# W10.5 Topology-Local Tributary Support-Load Screening

## Scope

W10.5 builds deterministic local vertical load paths from the reusable W10.1–W10.4 contracts and performs explicit simple-span tributary screening of gravity forces to qualified vertical supports.

The result is **screened support vertical force**. It is not a general piping support-reaction solution.

## Contracts

- `vertical-load-path-profile/v1`
- `vertical-load-path-model/v1`
- `tributary-support-load-screening/v1`
- `support-load-screening-audit/v1`

All outputs are deeply immutable and use deterministic semantic identities. No source/global chainage, timestamps or random IDs are used.

## Eligible topology

`SIMPLE_CHAIN_TRIBUTARY_VERTICAL_V1` accepts only a connected component that:

- is acyclic;
- has exactly two terminal ports;
- has no multi-port component or junction degree above two;
- can be ordered as one continuous component chain.

Orientation starts from the lexicographically smallest terminal port key. Stationing is local to the path in metres and is derived from approved component-port geometry.

Tee, cross, branch and loop topology remains blocked. Disconnected topology components remain separate path candidates.

## Support qualification

Support stations are derived from `support-attachment-model/v1` and `restraint-capability-model/v1`.

A support is eligible only when it has exactly one qualified attachment on the path, a deterministic station and vertical state `RESTRAINED`. Explicit and type-classified restraint bases remain distinguishishable.

Unattached, ambiguous, wrong-path, unavailable-station, duplicate-station, FREE, UNKNOWN, CONFLICT, GAP and SPRING records remain blocked. Coincident supports are not merged. No stiffness or displacement boundary is created.

## Load projection

Only `DISTRIBUTED_GRAVITY_LOAD` and `POINT_GRAVITY_LOAD` enter vertical screening. Each primitive must retain `GRAVITY_DOWN`, `globalVector: null`, a valid load case and geometry consistent with one qualified path.

`EXPLICIT_POINT_MOMENT` records are retained upstream but excluded from vertical-force screening.

EMPTY, OPE and HYD qualification remains independent. A blocked W10.4 case blocks only the corresponding path/case record.

## Tributary formulas

Adjacent qualified supports define simple screening spans.

Point-force formula:

```text
R_left  = P × (L − x) / L
R_right = P × x / L
```

Formula ID: `SIMPLE_SPAN_POINT_LOAD_REACTION_V1`.

Uniform distributed-segment formula:

```text
W = w × (b − a)
x_centroid = (a + b) / 2
R_left  = W × (L − x_centroid) / L
R_right = W × x_centroid / L
```

Formula ID: `SIMPLE_SPAN_UNIFORM_SEGMENT_REACTION_V1`.

Distributed primitives crossing support stations are split at each boundary before contribution calculation. Loads outside the first-to-last support envelope block the affected path/case as `OVERHANG_LOAD_UNSUPPORTED`.

## Equilibrium

`SUPPORT_FORCE_EQUILIBRIUM_CHECK_V1` compares screened applied gravity force with the sum of screened support vertical forces. The named profile exposes deterministic scale-aware absolute and relative tolerances. Engineering values are not rounded to force equilibrium.

## Workspace behavior

The Analysis Workspace provides:

- `Rebuild Vertical Load Paths`
- `Run Tributary Screening`
- `Export Support Load Screening`

Path health may rebuild after upstream contract changes. Screening never runs automatically. Upstream changes clear stale screening results while retaining the newest valid path health where possible.

The deterministic UTF-8 export includes the profile, path model, screening and audit, uses a dataset-derived filename and revokes its object URL.

Read-only APIs expose:

- `AnalysisWorkspace.getVerticalLoadPathModel()`
- `AnalysisWorkspace.getSupportLoadScreening()`
- `AnalysisWorkspace.getSupportLoadScreeningAudit()`

## Explicit exclusions

W10.5 does not implement global/source chainage, general route networks, branches, loops, overhang solutions, springs, stiffness matrices, displacement, rotation, thermal/pressure/wind/seismic loads, horizontal guide or line-stop forces, bending/shear diagrams, code stress, general reactions, viewport highlighting or reporting changes.
