MASTER RELEASE TITLE
--------------------
Simplified Analysis technical audit remediation and engineering workflow upgrade

MASTER RELEASE SUMMARY
----------------------
This release stack upgrades Simplified Analysis from a calculator/sketcher prototype into a more controlled engineering-screening workflow with:

1. Settings/defaults contract.
2. Unified engineering-data resolution.
3. Sketcher topology validation.
4. Solver result certification contract.
5. Active calculation reporting.
6. Browser workflow automation.
7. Full engineering-data migration.
8. Professional Sketcher drafting commands.
9. Branch-aware solver routing.
10. Report checker/export workflow.
11. Component master DB / fitting dimensions.
12. PCFX fitting roundtrip QA.
13. PCFX import/export UI and golden fixtures.

IMPORTANT ENGINEERING DISCLAIMER
--------------------------------
This stack improves traceability and QA but does not convert screening methods into final code-certified stress analysis.

Screening-only methods must remain labeled as:
- SCREENING
- BENCHMARKED_SCREENING
- BRANCH_SCREENING_ONLY

Final issue must remain blocked when:
- results are stale
- data is missing
- solver status is NOT_QUALIFIED or UNSUPPORTED_GEOMETRY
- component data is missing
- method is SCREENING and issue type is FINAL_ISSUE
- sample component DB rows are still SCREENING_SAMPLE

TOP RISKS
---------

RISK 1 — Hidden fallback values
Impact:
Wrong engineering result can appear valid.

Controls:
- V10 removes Sketcher pipe approximation.
- Missing data returns MISSING_DATA.
- Reports carry dataStatus.
- Issue workflow blocks missing data.

RISK 2 — Screening methods presented as final
Impact:
User may trust simplified result as code-certified stress analysis.

Controls:
- ENGINEERING_LEVEL.
- report issue type.
- SCREENING_LEVEL_NOT_FINAL blocker.
- branch solver returns BRANCH_SCREENING_ONLY.

RISK 3 — Branch topology pushed into GC3D
Impact:
GC3D formulas not valid for tee/olet branch systems.

Controls:
- GC3D branch rejection remains.
- V12 solver router routes branch to branch screening.
- GC3D includes routing hint.

RISK 4 — Tee branch misclassification
Impact:
Wrong main/branch path, wrong downstream stress/fitting interpretation.

Controls:
- vector-colinearity classification.
- PCFX stores tee classification metadata.
- V15/V16 fixtures verify branchSegment.

RISK 5 — Component DB sample data used as final
Impact:
Fitting dimensions/weights may not match project standard.

Controls:
- sourceStatus SCREENING_SAMPLE.
- issue workflow component-data blocker for missing/not-qualified.
- Future V17 should block SCREENING_SAMPLE for FINAL_ISSUE.

RISK 6 — Report generated from mock/demo data
Impact:
Wrong report issued.

Controls:
- activeReportContext.
- Reports tab no-active-report state.
- V9/V13 E2E tests assert no demo report.

RISK 7 — PCFX roundtrip data loss
Impact:
Edited fitting metadata lost during import/export.

Controls:
- rawAttributes, normalized, derived preserved.
- lossContract.
- V15 roundtrip validation.
- V16 golden fixtures.

LOSS CONTRACT
-------------
Unsupported records must not disappear.

lossContract item:
{
  severity,
  code,
  message,
  recordType,
  rawText,
  lineNumber,
  preservationAction
}

Preservation actions:
PRESERVED_RAW_ONLY
MAPPED_CANONICAL
DROPPED_UNSUPPORTED_WITH_WARNING
BLOCKED_UNSUPPORTED_REQUIRED

Parser rules:
1. Never invent fitting dimensions.
2. Never invent material data.
3. Preserve all unknown attributes in rawAttributes.
4. Unknown record types go to lossContract.
5. Required unsupported records block final issue.

Writer rules:
1. Write only fields that are known/mapped.
2. Preserve rawAttributes when available.
3. Emit warning if writer cannot reproduce unsupported raw record.
4. Roundtrip must be deterministic.
5. Do not write SCREENING_SAMPLE as VERIFIED.
